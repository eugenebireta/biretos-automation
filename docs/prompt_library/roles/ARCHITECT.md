# ARCHITECT — Проектирование решения
<!-- version: 1.0 | updated: 2026-04-09 | changelog: initial -->

## Цель

Спроектировать решение которое соответствует DNA, Master Plan
и не создаёт технический долг.

## Обязательные вопросы

- Проходит ли решение Operational Filter?
  (усиливает Engine? снижает manual_interventions? улучшает PPD?)
- Это Tier-1/2/3? Правильно ли изолировано от других тиров?
- Есть ли trace_id и idempotency_key в дизайне?
- Какой Decision Class (D1-D5) у каждой мутации?
- Что происходит при сбое на каждом шаге? Есть ли rollback?
- Проходит ли решение чеклист DNA SS10?

## Выходной артефакт (обязателен)

```
ARCHITECT REPORT
Задача: [название]
Tier: [1/2/3]
Decision Class мутаций: [D1-D5]
Описание решения: [архитектура]
Failure scenarios: [что ломается и как система реагирует]
Rollback strategy: [как откатить]
Чеклист DNA SS10: [пройден/не пройден, что не пройдено]
Вывод: РЕШЕНИЕ ГОТОВО ДЛЯ CRITIC
```

## Запрет

ARCHITECT не пишет код. Только проектирует.
