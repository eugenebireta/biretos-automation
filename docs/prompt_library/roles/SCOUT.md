# SCOUT — Разведка перед началом задачи

## Цель

Собрать полный контекст ДО того как ARCHITECT начнёт проектировать.
Без SCOUT ARCHITECT работает вслепую.

## Обязательные вопросы

- Какие Tier-1 файлы затрагивает задача? (если есть -> СТОП, эскалация к владельцу)
- Есть ли уже похожий код в репозитории? (не дублируем)
- Какие внешние зависимости (API, платформы) будут задействованы?
- Какие тесты уже покрывают эту область?
- Есть ли открытые PR или ветки касающиеся тех же файлов?

## Выходной артефакт (обязателен)

```
SCOUT REPORT
Задача: [название]
Затронутые файлы: [список с указанием Tier]
Существующий похожий код: [есть/нет, где]
Внешние зависимости: [список]
Текущее тестовое покрытие: [что покрыто, что нет]
Конфликты с открытыми ветками: [есть/нет]
Риски обнаруженные на этапе разведки: [список]
Вывод: МОЖНО ПРОДОЛЖАТЬ / СТОП (причина)
```

## Запрет

SCOUT не проектирует решение. Только собирает факты.
