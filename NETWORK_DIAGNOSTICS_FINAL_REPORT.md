## 4. Что нужно сделать для полной диагностики

### Шаг 1: Тесты на VPS (приоритет 1)

Выполнить на VPS через SSH:

```bash
# 1. Traceroute
traceroute -n -w 1 chatgpt.com > traceroute_vps.txt

# 2. MTR (если установлен)
mtr --report --report-cycles 50 --no-dns chatgpt.com > mtr_vps.txt

# 3. Ping для jitter (200 пакетов)
ping -c 200 -i 1 chatgpt.com > ping_vps.txt

# 4. Проверка HTTPS
curl -v --max-time 10 https://chatgpt.com 2>&1 | tee https_vps.txt
```

**Ожидаемый результат:** Получить метрики RTT, jitter, потери пакетов с VPS.

---

### Шаг 2: Определить статус VPN (приоритет 2)

Проверить:
- Активен ли VPN/прокси в данный момент
- Какие сетевые интерфейсы активны
- Куда идет трафик (через VPN или напрямую)

**Команды:**
```powershell
# Проверить активные VPN интерфейсы
Get-NetAdapter | Where-Object {$_.InterfaceDescription -like "*VPN*" -or $_.InterfaceDescription -like "*TAP*" -or $_.InterfaceDescription -like "*TUN*"}

# Проверить маршруты
Get-NetRoute -DestinationPrefix 0.0.0.0/0

# Проверить прокси настройки
netsh winhttp show proxy
```

**Ожидаемый результат:** Понять, используется ли VPN/прокси сейчас.

---

### Шаг 3: Ручное тестирование в браузере (приоритет 3)

1. **Измерение RTT через DevTools:**
   - Открыть chatgpt.com
   - F12 → Network tab
   - Отправить сообщение в ChatGPT
   - Зафиксировать время ответа (Time column)
   - Повторить 10 раз, вычислить avg/min/max

2. **Idle тест:**
   - Открыть диалог в ChatGPT
   - Оставить на 5-10 минут без активности
   - Отправить сообщение после паузы
   - Зафиксировать: работает/переподключение/ошибка

3. **Сравнение:**
   - Выполнить тесты с VPN
   - Выполнить тесты без VPN
   - Сравнить метрики

**Ожидаемый результат:** Получить реальные метрики работы ChatGPT UI.
