# Чеклист миграции медиа в Shopware 6

## ⚠️ КРИТИЧЕСКИ ВАЖНО

### Проблема с media_type

**Медиа, загруженные через API (`/api/media` + `/api/_action/media/{id}/upload`), имеют `media_type = NULL` в БД.**

Это приводит к тому, что:
- ❌ Shopware **навсегда пропускает** эти медиа при генерации thumbnails
- ❌ Thumbnails не создаются
- ❌ Превьюшки не отображаются в каталоге

## ✅ ОБЯЗАТЕЛЬНЫЙ ПОРЯДОК ДЕЙСТВИЙ

### После загрузки медиа через API

**ВСЕГДА выполнять в указанном порядке:**

1. **`media:generate-media-types`** — ОБЯЗАТЕЛЬНО ПЕРВЫМ
   - Заполняет поле `media_type` для всех медиа
   - Анализирует `mime_type` и определяет тип медиа
   - Без этого шага thumbnails НЕ будут генерироваться

2. **`media:generate-thumbnails`** — ТОЛЬКО ПОСЛЕ ШАГА 1
   - Генерирует thumbnails для всех медиа
   - Работает только если `media_type` установлен
   - Создаёт файлы на диске в `/public/media/thumbnail/`

3. **`cache:clear`** — ОПЦИОНАЛЬНО
   - Очищает кеш Shopware
   - Рекомендуется после генерации thumbnails

## 🚫 ЧТО НЕ ДЕЛАТЬ

### ❌ Запрещено:

- **Ручное исправление через БД** — недопустимо, нарушает целостность данных
- **Попытки генерировать thumbnails без media_type** — бесполезно, Shopware пропустит
- **Пропуск шага `media:generate-media-types`** — критическая ошибка
- **Изменение thumbnail sizes** — не решает проблему
- **Изменение storefront/тем** — не связано с проблемой

## 📋 АВТОМАТИЧЕСКАЯ ОБРАБОТКА

### Скрипт `add_media_to_products.py`

Скрипт **автоматически** выполняет все необходимые шаги:

1. ✅ Загружает медиа через API (золотой стандарт Shopware)
2. ✅ **Автоматически** выполняет `media:generate-media-types` после загрузки
3. ✅ **Автоматически** выполняет `media:generate-thumbnails` после инициализации types
4. ✅ Проверяет результат и выводит статистику

**Не требуется ручное вмешательство!**

## 🔍 ПРОВЕРКА РЕЗУЛЬТАТА

### После выполнения скрипта проверьте:

1. **Media types установлены:**
   ```sql
   SELECT COUNT(*) FROM media WHERE media_type IS NULL AND mime_type LIKE 'image/%';
   ```
   Должно быть: `0`

2. **Thumbnails созданы:**
   ```bash
   docker exec shopware find /var/www/html/public/media/thumbnail -type f | wc -l
   ```
   Должно быть: `> 0` (количество зависит от количества медиа и thumbnail sizes)

3. **Thumbnails в БД:**
   ```sql
   SELECT COUNT(*) FROM media_thumbnail;
   ```
   Должно быть: `> 0`

## ⚡ БЫСТРАЯ СПРАВКА

| Проблема | Решение |
|----------|---------|
| Thumbnails не генерируются | Выполнить `media:generate-media-types` ПЕРВЫМ |
| Media пропускаются при генерации | Проверить, что `media_type IS NOT NULL` |
| Превьюшки не отображаются | Проверить наличие thumbnails на диске |
| Ошибка при генерации | Проверить логи, убедиться что media types установлены |
