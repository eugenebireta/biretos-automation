# Быстрая выгрузка (Snapshot) товаров InSales

Скрипт для экстренной выгрузки всех товаров из InSales API перед возможным отключением сервиса.

## Возможности

- ✅ **Параллельная загрузка** с контролем rate limit (настраиваемое количество потоков)
- ✅ **Retry/Backoff** на ошибки 429 и 5xx с exponential backoff
- ✅ **Checkpoint механизм** - можно продолжить после сбоя
- ✅ **Два формата вывода**: NDJSON (каждый товар на строку) и CSV (минимальный набор полей)
- ✅ **Прогресс-логирование** каждые 100 товаров
- ✅ **Защита от дубликатов** - пропускает уже скачанные товары при продолжении

## Быстрый старт

```bash
cd insales_to_shopware_migration/src
python snapshot_products.py
```

Скрипт использует конфигурацию из `config.json` (путь можно указать через `--config`).

## Параметры командной строки

```bash
python snapshot_products.py [OPTIONS]

Опции:
  --config PATH           Путь к config.json (по умолчанию: ../config.json)
  --output PATH           Директория для сохранения snapshot (по умолчанию: ../insales_snapshot)
  --per-page N            Товаров на страницу (по умолчанию: 250, максимум обычно 250)
  --workers N             Количество параллельных потоков (по умолчанию: 5)
  --max-retries N         Максимальное количество попыток для одной страницы (по умолчанию: 5)
  --retry-delay SECONDS   Базовая задержка для retry в секундах (по умолчанию: 2.0)
  --timeout SECONDS       Timeout для HTTP запросов (по умолчанию: 30)
  --reset-checkpoint      Игнорировать checkpoint и начать заново
```

## Примеры использования

### Стандартная выгрузка

```bash
python snapshot_products.py
```

### Быстрая выгрузка с увеличенным параллелизмом

```bash
python snapshot_products.py --workers 10 --per-page 250
```

### Продолжить после сбоя

Если выгрузка была прервана, просто запустите скрипт снова - он автоматически продолжит с последней успешно загруженной страницы:

```bash
python snapshot_products.py --config config.json --output insales_snapshot
```

### Начать заново (сброс checkpoint)

```bash
python snapshot_products.py --reset-checkpoint
```

## Структура выходных файлов

После выполнения в указанной директории (`--output`) будут созданы:

```
insales_snapshot/
├── products.ndjson       # Все товары в формате NDJSON (один товар на строку)
├── products.csv          # CSV с минимальным набором полей (id, sku, name, price, category_id, category_path)
└── checkpoint.json       # Состояние выгрузки для продолжения
```

### Формат NDJSON

Каждая строка - валидный JSON объект с полными данными товара из InSales API.

```json
{"id": 12345, "title": "Название товара", "variants": [...], ...}
{"id": 12346, "title": "Другой товар", "variants": [...], ...}
```

### Формат CSV

Таблица с минимальным набором полей для быстрого просмотра:

```csv
id,sku,name,price,category_id,category_path
12345,SKU001,Название товара,1000.00,42,42
```

## Особенности работы

### Параллелизм и Rate Limit

Скрипт использует ThreadPoolExecutor для параллельной загрузки страниц. По умолчанию используется 5 потоков - это безопасное значение для большинства API. При необходимости можно увеличить до 10, но не рекомендуется больше, чтобы не получить rate limit.

### Retry логика

- **429 (Rate Limit)**: Exponential backoff (2s, 4s, 8s, 16s, 32s)
- **5xx (Server Error)**: Exponential backoff с теми же задержками
- **401/403/404**: Не retry (сразу пропускается)
- **Timeout**: Retry с backoff

### Checkpoint механизм

Checkpoint сохраняется после каждой успешно загруженной страницы и содержит:
- `last_page`: Последняя успешно загруженная страница
- `downloaded_count`: Количество уже скачанных товаров
- `failed_pages`: Список страниц, которые не удалось загрузить
- `total_pages`: Общее количество страниц (если известно)

При продолжении:
- Пропускаются все страницы до `last_page`
- Повторно пытаются загрузить страницы из `failed_pages`
- Загружаются только новые страницы

### Защита от дубликатов

При продолжении с checkpoint скрипт загружает ID всех товаров из существующего NDJSON файла и пропускает товары с такими же ID при обработке новых страниц.

## Рекомендации

1. **Для максимальной скорости**: Используйте `--workers 10 --per-page 250`
2. **Для стабильности**: Используйте значения по умолчанию
3. **При частых 429 ошибках**: Уменьшите `--workers` до 3
4. **При нестабильном соединении**: Увеличьте `--timeout` до 60

## Устранение проблем

### Скрипт останавливается с ошибками 429

Уменьшите количество потоков:
```bash
python snapshot_products.py --workers 3
```

### Некоторые страницы не загружаются

Проверьте список неудачных страниц в финальном выводе и запустите скрипт снова - он попытается загрузить их повторно.

### Checkpoint повреждён

Удалите файл `checkpoint.json` или используйте `--reset-checkpoint` для начала заново.

## Команда для повторного запуска

После завершения (или прерывания) скрипт выводит команду для повторного запуска:

```
Для продолжения запустите:
  python snapshot_products.py --config config.json --output insales_snapshot
```
