# Миграция категорий InSales → Shopware 6

## Описание

Скрипт `migrate_categories.py` импортирует категории из InSales в Shopware 6 с правильной структурой навигации.

### Особенности

- ✅ **Идемпотентность**: Повторный запуск не создает дублей (используется `migration_map.json`)
- ✅ **Пропуск "Каталога"**: Служебная категория "Каталог" из InSales не импортируется
- ✅ **Усыновление детей**: Дети категории "Каталог" становятся прямыми детьми `ROOT_NAV`
- ✅ **Верхнеуровневые категории**: Все категории без родителя в InSales → прямые дети `ROOT_NAV`
- ✅ **Dry Run режим**: Проверка структуры без реальных изменений
- ✅ **Сохранение видимости**: Статус `active` берется из `is_hidden` InSales

## Требования

1. **ROOT_NAV категория**: В Shopware должна быть создана техническая категория `ROOT_NAV` (inactive, invisible)
2. **Конфигурация**: В `config.json` должен быть указан `root_nav_id`

## Настройка

### 1. Создайте ROOT_NAV в Shopware

В админке Shopware:
1. Перейдите в **Catalogues → Categories**
2. Создайте новую категорию с названием `ROOT_NAV`
3. Настройки:
   - **Active**: ❌ (inactive)
   - **Visible in navigation**: ❌ (invisible)
4. Скопируйте **ID** категории (например: `01994d23acb670fa926de3796d173b74`)

### 2. Обновите config.json

Добавьте `root_nav_id` в секцию `shopware`:

```json
{
  "shopware": {
    "url": "https://your-shopware.com",
    "access_key_id": "...",
    "secret_access_key": "...",
    "root_category_id": "...",
    "root_nav_id": "01994d23acb670fa926de3796d173b74"
  }
}
```

**Важно**: `root_nav_id` — это ID вашей технической категории `ROOT_NAV`.

## Использование

### Dry Run (рекомендуется сначала)

Проверка структуры без реальных изменений:

```bash
cd insales_to_shopware_migration/src
python migrate_categories.py --dry-run
```

Вывод покажет:
- Дерево категорий, которое будет создано
- Статистику (сколько новых, сколько уже существует)
- Какие категории будут пропущены

### Реальный импорт

После проверки dry run запустите реальный импорт:

```bash
python migrate_categories.py
```

Скрипт:
1. Прочитает структуру из `logs/insales_structure.json`
2. Пропустит категорию "Каталог"
3. Создаст/обновит категории в Shopware
4. Сохранит маппинг в `migration_map.json`

### Параметры командной строки

```bash
python migrate_categories.py [OPTIONS]

Опции:
  --config PATH      Путь к config.json (по умолчанию: ../config.json)
  --structure PATH   Путь к insales_structure.json (по умолчанию: ../logs/insales_structure.json)
  --map PATH         Путь к migration_map.json (по умолчанию: ../migration_map.json)
  --dry-run          Режим проверки без изменений
  -h, --help         Показать справку
```

## Логика работы

### Определение родителя категории

1. **Верхнеуровневая** (нет `parent_id` в InSales) → `ROOT_NAV`
2. **Ребенок "Каталога"** (`parent_id` = ID "Каталога") → `ROOT_NAV`
3. **Обычная категория** → сохраняет иерархию (рекурсивно создается родитель)

### Пропуск "Каталога"

Категория с названием **"Каталог"** (точное совпадение) не импортируется. Все её дети "усыновляются" к `ROOT_NAV`.

### Видимость

- `active` в Shopware = `not is_hidden` из InSales
- Если в InSales категория скрыта (`is_hidden: true`), в Shopware она будет неактивной

## Идемпотентность

Скрипт использует файл `migration_map.json` для отслеживания уже импортированных категорий:

```json
{
  "categories": {
    "21094370": "fc027b71175946199e578a53ae5ce8da",
    "21197153": "0f1349cbb19741a6ad4732f0949f9052"
  }
}
```

- Ключ: ID категории в InSales
- Значение: UUID категории в Shopware

При повторном запуске:
- Категории, уже существующие в маппинге, **не создаются заново**
- Новые категории добавляются в маппинг

## Устранение проблем

### Ошибка: "root_nav_id is not defined"

**Решение**: Добавьте `root_nav_id` в `config.json` (см. раздел "Настройка").

### Дубли категорий в меню

**Причина**: Возможно, `migration_map.json` был очищен или скрипт запускался несколько раз без сохранения маппинга.

**Решение**: 
1. Проверьте `migration_map.json` — там должны быть записи для всех категорий
2. Если дубли уже созданы, удалите их вручную в админке Shopware
3. Запустите dry run для проверки

### Категории не появляются в меню

**Проверьте**:
1. В Sales Channel → Entry point main navigation установлена категория `ROOT_NAV`
2. Категории имеют `active: true` и `visible: true` в Shopware
3. В админке Shopware проверьте структуру дерева категорий

## Следующие шаги

После успешного импорта категорий:

1. **Проверьте структуру** в админке Shopware (Catalogues → Categories)
2. **Настройте Sales Channel**: Sales Channels → ваш магазин → Entry point = `ROOT_NAV`
3. **Проверьте меню** на сайте
4. **Импортируйте товары** (используя `migration_map.json` для привязки к категориям)

## Файлы

- `src/migrate_categories.py` — основной скрипт
- `config.json` — конфигурация (должен содержать `root_nav_id`)
- `logs/insales_structure.json` — структура данных InSales
- `migration_map.json` — маппинг ID (создается/обновляется автоматически)
