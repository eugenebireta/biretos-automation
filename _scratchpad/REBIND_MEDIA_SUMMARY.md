# Сводка: Скрипт перепривязки медиа

## ✅ СОЗДАН СКРИПТ

**Файл:** `_scratchpad/rebind_product_media.py`

## 🎯 НАЗНАЧЕНИЕ

Перепривязка существующих медиа к товарам через Shopware API для триггера генерации thumbnails.

## 📋 РЕАЛИЗОВАНО

### 1. Логика получения медиа

- ✅ Использует `product-media` endpoint (наиболее надёжный способ)
- ✅ Получает `mediaId` и `position` для каждого медиа
- ✅ Сортирует по `position`
- ✅ Fallback на `relationships` если `product-media` не работает

### 2. Логика перепривязки

- ✅ Подготавливает payload с теми же `mediaId` и `position`
- ✅ НЕ меняет `coverId`
- ✅ НЕ добавляет новых полей
- ✅ Отправляет `PATCH /api/product/{id}`

### 3. Батчинг и безопасность

- ✅ Обрабатывает товары пакетами по 50 штук
- ✅ Задержка 1 секунда между батчами (rate limit friendly)
- ✅ Однопоточная обработка
- ✅ При ошибке товара продолжает работу

### 4. Логирование

- ✅ `[SKIP]` — если медиа нет
- ✅ `[OK]` — productNumber → media rebound (количество медиа)
- ✅ `[FAIL]` — productNumber → reason
- ✅ Прогресс каждые 50 товаров
- ✅ Итоговая статистика

### 5. Идемпотентность

- ✅ Повторный запуск не создаёт дубликатов
- ✅ Не меняет данные товара (переотправляет те же данные)

## 🚫 ОГРАНИЧЕНИЯ (соблюдены)

- ❌ НЕ загружает файлы заново
- ❌ НЕ создаёт новые media
- ❌ НЕ трогает thumbnail sizes
- ❌ НЕ лезет в БД напрямую
- ✅ Работает ТОЛЬКО через Shopware API

## 📝 ИСПОЛЬЗОВАНИЕ

```bash
python _scratchpad/rebind_product_media.py
```

## ⚠️ ВАЖНО

Скрипт **НЕ вызывает** генерацию thumbnails автоматически.

После выполнения запустите вручную:
```bash
docker exec shopware php bin/console media:generate-thumbnails
```

## 📊 ОЖИДАЕМЫЙ РЕЗУЛЬТАТ

После выполнения скрипта и генерации thumbnails:
- ✅ Shopware заново поставит media в очередь обработки
- ✅ Thumbnails начнут появляться на диске
- ✅ Превьюшки появятся в листингах
- ✅ Превьюшки появятся в карточках товаров
- ✅ Превьюшки появятся в админке

## 🔧 ТЕХНИЧЕСКИЕ ДЕТАЛИ

### Формат payload

```python
{
    "id": "<productId>",
    "media": [
        {"mediaId": "<mediaId1>", "position": 1},
        {"mediaId": "<mediaId2>", "position": 2}
    ]
}
```

### Параметры батчинга

- `BATCH_SIZE = 50` — размер батча
- `BATCH_DELAY = 1.0` — задержка между батчами (секунды)

### Обработка ошибок

- При ошибке получения медиа → пропуск товара
- При ошибке PATCH → логирование, продолжение работы
- Все ошибки собираются в список для итогового отчёта
