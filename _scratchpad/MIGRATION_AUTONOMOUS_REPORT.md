# Отчёт по автономному исправлению ошибки "No price for default currency defined"

## Дата: 2025-12-09

## Проблема
Shopware API возвращает ошибку `FRAMEWORK__WRITE_CONSTRAINT_VIOLATION: No price for default currency defined` при попытке создать товар, даже когда в payload присутствует цена для системной валюты.

## Выполненные шаги

### 1. Определение default currency

**Попытка 1: System Config**
- Запрос: `GET /api/system-config?domain=core.currency`
- Результат: Ключ `core.currency.defaultCurrency` не найден в System Config
- Найдено 67 записей в System Config, но нет настроек валюты по умолчанию

**Попытка 2: Sales Channels**
- Запрос: `GET /api/sales-channel` и `POST /api/search/sales-channel`
- Результат: Найдено 2 Sales Channel, но у обоих `currencyId = None`
- Sales Channel ID: `01994d25627d7303ba92b97e96becd52`, `98432def39fc4624b33213a56b8c944d`

**Попытка 3: Системная валюта через GET /api/currency**
- Метод: `get_system_currency_id()` - берёт первую валюту из списка
- Результат: `01994d23e86b720fa0e16dd0348f48c6`
- Проблема: Shopware не распознаёт эту валюту как default currency

**Попытка 4: Поиск валюты с isSystemDefault**
- Запрос: `POST /api/search/currency` с фильтром `isSystemDefault = true`
- Результат: Ошибка - `isSystemDefault` является Runtime полем и не может использоваться в критериях поиска

**Итоговый defaultCurrencyId**: `01994d23e86b720fa0e16dd0348f48c6` (первая валюта из списка)

### 2. Валидация payload

**Текущий формат payload.price:**
```json
[
  {
    "currencyId": "01994d23e86b720fa0e16dd0348f48c6",
    "gross": 455.0,
    "net": 455.0,
    "linked": false
  }
]
```

**Валидация добавлена:**
- Проверка наличия системной валюты перед созданием payload
- Проверка наличия цены для системной валюты в массиве price
- Системная валюта всегда добавляется первой в массив price

**Результат**: Payload формируется корректно, валидация проходит успешно.

### 3. Проверка влияния контекста запроса

**Попытка A: Без заголовков**
- Результат: Ошибка "No price for default currency defined"

**Попытка B: С заголовком `sw-currency-id`**
- Заголовок: `sw-currency-id: 01994d23e86b720fa0e16dd0348f48c6`
- Результат: Ошибка сохраняется

**Попытка C: С заголовками `sw-currency-id` и `sw-sales-channel-id`**
- Заголовки: 
  - `sw-currency-id: 01994d23e86b720fa0e16dd0348f48c6`
  - `sw-sales-channel-id: 01994d25627d7303ba92b97e96becd52`
- Результат: Ошибка сохраняется

**Попытка D: Прямой POST вместо Sync API**
- Endpoint: `POST /api/product` вместо `POST /api/_action/sync`
- Результат: Ошибка сохраняется

### 4. Логирование запросов

**Логируемые данные:**
- Request URL: `https://77.233.222.214/api/_action/sync` или `/api/product`
- Headers: `sw-currency-id=01994d23e86b720fa0e16dd0348f48c6`
- Payload price: Массив с одной записью для системной валюты
- Response: Ошибка 400 с деталями

**Наблюдение**: Payload формируется правильно, заголовки устанавливаются, но Shopware всё равно не распознаёт валюту как default.

## Анализ проблемы

### Возможные причины:

1. **Shopware определяет default currency через другой механизм**

   - Возможно, Shopware не настроен правильно
