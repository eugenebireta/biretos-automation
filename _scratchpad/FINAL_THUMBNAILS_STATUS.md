### Почему thumbnails не генерируются:

Команда `media:generate-thumbnails` выполняется (exit status 0), но:
- Не выводит статистику (Generated/Skipped/Errors)
- Не создаёт файлы на диске
- Не показывает ошибки

**Возможные причины:**
1. Shopware использует оригинальные изображения для превьюшек (ресайз на лету)
2. Thumbnails генерируются асинхронно через очередь (нужно проверить messenger)
3. Проблема в настройках темы/шаблона каталога

## Решение

### Вариант 1: Shopware использует оригинальные изображения (наиболее вероятно)

В Shopware 6.7.x для превьюшек в каталоге могут использоваться оригинальные изображения с автоматическим ресайзом на лету, а не thumbnails. Это нормальное поведение.

**Проверка:**
1. Откройте каталог: https://dev.bireta.ru/navigation/fc027b71175946199e578a53ae5ce8da
2. Откройте исходный код страницы (F12)
3. Найдите изображения товаров
4. Проверьте URL изображений - если они указывают на оригинальные файлы (не `/thumbnail/...`), то это нормально

**Если изображения не отображаются вообще:**
- Проблема в шаблоне каталога или настройках темы
- Нужно проверить шаблон `listing.html.twig` или настройки темы

### Вариант 2: Проверка очереди Shopware

```bash
# Проверка очереди
ssh root@77.233.222.214 "docker exec shopware php bin/console messenger:stats"

# Запуск обработки очереди
ssh root@77.233.222.214 "docker exec shopware php bin/console messenger:consume async -vv"
```

### Вариант 3: Проверка шаблона каталога

Если превьюшки не отображаются, проблема может быть в шаблоне каталога:

1. Войдите в админку: `https://77.233.222.214/admin`
2. Перейдите: **Settings → Themes**
3. Выберите активную тему
4. Проверьте настройки отображения изображений в каталоге
5. Убедитесь, что включено отображение изображений товаров

## Рекомендации

### Если изображения не отображаются в каталоге:

1. **Проверьте шаблон каталога** - возможно, он не настроен на отображение изображений
2. **Проверьте настройки темы** - убедитесь, что включено отображение изображений
3. **Проверьте права доступа** к файлам медиа на сервере
4. **Проверьте логи Shopware** на наличие ошибок

### Если изображения отображаются, но без превьюшек:

Это нормально - Shopware может использовать оригинальные изображения с ресайзом на лету. Thumbnails используются для оптимизации производительности, но не обязательны для отображения.
