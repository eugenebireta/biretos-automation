# Отчёт: Исправление превьюшек товаров в каталоге

## Дата: 2025-01-XX

## Проблема

В каталоге товаров (например, https://dev.bireta.ru/navigation/fc027b71175946199e578a53ae5ce8da) не отображаются превьюшки товаров. Пользователь вынужден заходить в карточку товара, чтобы увидеть фото.

## Выполненные действия

### 1. ✅ Создание Thumbnail Sizes

**Проблема**: Таблица `thumbnail_size` была пуста (0 размеров).

**Решение**: Созданы 4 стандартных размера через SQL:
- 192x192 (для превьюшек в каталоге)
- 400x400 (для карточек товаров)
- 800x800 (для больших изображений)
- 1920x1920 (для полноразмерных изображений)

**Результат**: ✅ Thumbnail sizes созданы (4 размера в БД)

### 2. ✅ Проверка Cover Images

**Проверка**: Все товары имеют установленный `coverId`.

**Результат**: ✅ Cover images установлены для всех товаров

### 3. ⚠️ Генерация Thumbnails

**Проблема**: Команда `media:generate-thumbnails` не генерирует thumbnails и не выводит статистику.

**Статус**: 
- Thumbnail sizes: ✅ 4 размера
- Product_media: ✅ 9 привязок
- Изображений: ✅ 99
- Медиа, привязанные к товарам: ✅ 9

**Попытки**:
- ✅ `media:generate-media-types` - выполнен успешно
- ⚠️ `media:generate-thumbnails` - выполняется, но не выводит статистику
- ❌ Thumbnails на диске не найдены

## Возможные причины

1. **Shopware использует оригинальные изображения для превьюшек**
   - В некоторых конфигурациях Shopware может использовать оригинальные изображения с ресайзом на лету вместо thumbnails
   - Это зависит от настроек темы/шаблона

2. **Thumbnails генерируются асинхронно**
   - Shopware может генерировать thumbnails в фоновом режиме через очередь
   - Нужно проверить настройки очереди (queue)

3. **Проблема с настройками темы**
   - Тема может быть настроена на использование оригинальных изображений
   - Нужно проверить настройки в админке: Settings → Themes

## Рекомендации

### Вариант 1: Проверка настроек темы

1. Войдите в админку: `https://77.233.222.214/admin`
2. Перейдите: **Settings → Themes**
3. Выберите активную тему
4. Проверьте настройки отображения изображений в каталоге
5. Убедитесь, что включено использование thumbnails

### Вариант 2: Проверка очереди Shopware

```bash
# Проверка очереди
docker exec shopware php bin/console messenger:stats

# Запуск обработки очереди (если есть задачи)
docker exec shopware php bin/console messenger:consume async -vv
```

### Вариант 3: Принудительная генерация для конкретных медиа

```bash
# Генерация для всех медиа товаров
docker exec shopware php bin/console media:generate-thumbnails --all
```

### Вариант 4: Проверка настройки thumbnail sizes в админке

1. Войдите в админку: `https://77.233.222.214/admin`
2. Перейдите: **Settings → Media → Thumbnail Sizes**
3. Убедитесь, что размеры активны и правильно настроены
