# Pre-Flight Check Rules — Autonomous Mode

## 1. Перечень обязательных проверок
- Убедись, что проект находится в `C:\cursor_project\biretos-automation`.
- Загрузи `.env`, `.env.n8n`, `env_keys_READY_TO_RENAME`, `config/secrets.json`, если требуется.
- Проверь наличие критичных переменных:
  - `OPENROUTER_API_KEY`
  - SSH креды (ключ или пароль к `root@216.9.227.124`)
  - n8n API key / URL
  - Shopware creds (если задача про экспорт)
  - DB параметры (если задействованы)

## 2. Автопоиск ключей
1. Поищи ключи в следующих локациях в указанном порядке:
   - `.env`
   - `.env.n8n`
   - `env_keys_READY_TO_RENAME`
   - `config/secrets.json`
2. Если ключ найден — не спрашивай пользователя, просто используй.
3. Если ключ не найден — зафиксируй попытку в лог и только тогда спроси.

## 3. Проверка инфраструктуры (минимальная)
- `Test-Connection 216.9.227.124 -Count 1`
- `Invoke-WebRequest http://localhost:5678` (n8n)
- Если задача связана с Shopware — проверить `https://shopware.biretos.ae` (HEAD запрос)
- Если задача связана с T-Bank — убедиться, что токены свежие (смотри `brand-catalog-automation/diagnostics/tbank_*`)
- Если операция требует SSH/SCp — перед началом пингануть соответствующий сервер из `servers.json` (по `Host`). Если недоступен — не пытаться подключаться, сразу сообщить пользователю.

## 4. Поведение при отсутствии ключей
- Сделай ещё одну попытку автопоиска (включая поиск по памяти пользователя)
- Если все варианты исчерпаны — сообщи пользователю: какой ключ нужен и где уже искал.

## 5. Отчёт о Pre-Flight
Перед выполнением плана покажи краткий отчёт:
```
Pre-Flight Check:
- Env: OK/FAIL
- Keys: перечисли, какие ключи найдены
- VPS: Online/Offline
- n8n: Online/Offline
```

## 6. Гигиена `_scratchpad`
- Если `_scratchpad/` содержит более 50 файлов — перед стартом работы запусти
  `pwsh infrastructure/scripts/Clean-Workspace.ps1` и приложи результат в отчёт.
