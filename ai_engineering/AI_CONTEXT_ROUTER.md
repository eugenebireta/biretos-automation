# 🧭 AI Context Router — Meta-Router для выбора контекста

**СТАТУС:** Meta-router для выбора правил по задаче. Не является механизмом, правилом или gate.

**НАЗНАЧЕНИЕ:** Этот файл определяет, **ЧТО читать** из системы AI-правил в зависимости от задачи. Используется для экономии токенов и узкой загрузки контекста вместо чтения всех правил подряд.

**КРИТИЧЕСКОЕ ПРАВИЛО:** Под фразой пользователя "правила работы" понимается вся система правил проекта, но загружается **ТОЛЬКО** через этот роутер, а не все файлы подряд.

---

## MINIMAL AUTOLOAD SET

**Базовый набор (4-5 файлов), загружаемый ВСЕГДА:**

1. **`ai_engineering/error_protocol_trigger.txt`**
   - Высший приоритет: "overrides all other instincts. No exceptions."
   - Критичен для безопасности при любых ошибках
   - Всегда активен, независимо от задачи

2. **`ai_engineering/CONTEXT_RULES.md`** (или `ask_context_wrapper.txt` как краткая версия)
   - Базовое правило Controlled Context Mode
   - Запрет автоматического сканирования проекта
   - Фундаментальное ограничение для всех задач

3. **`ai_engineering/AI_RULES_INDEX.md`**
   - Навигационный индекс для контекст-роутинга
   - Карта всех правил и их ролей
   - Используется для выбора ON-DEMAND групп

4. **`ai_engineering/ask_behavior_rules.txt`** (базовые правила ASK)
   - Минимальные правила поведения ASK
   - Формирование плана, запрос подтверждения, запрет выполнения
   - Необходимо для корректной работы ASK

5. **Ссылка на `ai_engineering/MECHANISM_HIERARCHY.md`** (on-demand)
   - Архитектурный gate для новых механизмов
   - Загружается только при обнаружении намерения создать механизм
   - Ссылка присутствует в MINIMAL SET, но сам файл загружается по требованию

**Итого:** 4 файла всегда в памяти + 1 по требованию.

## ROUTER ALGORITHM

**Алгоритм выбора группы по задаче:**

### Шаг 1: Начало сессии

1. Загрузить **MINIMAL AUTOLOAD SET** (4-5 файлов)
2. Прочитать `AI_RULES_INDEX.md` для навигации
3. **ЗАПРЕТ:** Не читать другие файлы правил без явной необходимости

### Шаг 2: Анализ запроса пользователя

1. Извлечь ключевые слова из запроса пользователя
2. Определить категорию задачи по триггерам (сопоставить с ON-DEMAND GROUPS)
3. Если категория не определена — использовать **"planning"** по умолчанию для ASK
4. Если запрос содержит фразу "правила работы" — использовать этот роутер для выбора, а не загружать все файлы

### Шаг 3: Выбор ON-DEMAND группы

1. Сопоставить ключевые слова с триггерами групп
2. Если найдено совпадение — загрузить соответствующую группу
3. Если найдено несколько совпадений — загрузить объединение групп (максимум 2-3 группы за сессию)
4. **ЗАПРЕТ:** Загружать более 3 групп одновременно (приоритизировать по релевантности)

### Шаг 4: Проверка обязательных зависимостей

1. Если задача требует выполнения (AGENT) — обязательно загрузить GROUP "execution"
2. Если задача требует проверки окружения — обязательно загрузить GROUP "preflight"
3. Если обнаружена ошибка — обязательно загрузить GROUP "debug" (через error_protocol_trigger.txt)
4. Если обнаружено намерение создать механизм — обязательно загрузить GROUP "architecture" (MECHANISM_HIERARCHY.md)

### Шаг 5: Запрет чтения "всего подряд"

1. **ЗАПРЕТ:** Автоматическое чтение файлов правил вне выбранных групп
2. **ЗАПРЕТ:** Чтение директорий `ai_engineering/` без явного указания файла
3. **ЗАПРЕТ:** Загрузка всех правил "на всякий случай"
4. **РАЗРЕШЕНИЕ:** Чтение только через явную ссылку в `AI_RULES_INDEX.md` или по выбору роутера

### Шаг 6: Динамическая загрузка при необходимости

1. Если в процессе работы требуется правило из другой группы — загрузить только этот файл
2. Зафиксировать причину загрузки (например, "требуется для проверки механизма")
3. **ОГРАНИЧЕНИЕ:** Максимум 2-3 дополнительных файла за сессию

### Шаг 7: Кэширование загруженных групп

1. В рамках одной сессии не перезагружать уже загруженные группы
2. При переходе в AGENT режим — добавить GROUP "execution" к уже загруженным
3. При обнаружении ошибки — добавить GROUP "debug" к уже загруженным

**ЗАПРЕТ:** Загрузка всех 17+ файлов правил при запросе "правила работы". Это нарушение экономии токенов и Controlled Context Mode.

## Зависимости между группами

**Явные зависимости (загружать автоматически):**

1. **GROUP "execution"** требует:
   - GROUP "planning" (если не загружена, загрузить для контекста плана)
   - MINIMAL AUTOLOAD SET (всегда)

2. **GROUP "architecture"** требует:
   - MINIMAL AUTOLOAD SET (всегда)
   - Ссылка на `MECHANISM_HIERARCHY.md` (загружается автоматически)

3. **GROUP "debug"** требует:
   - MINIMAL AUTOLOAD SET (всегда)
   - `error_protocol_trigger.txt` (уже в MINIMAL SET)

4. **GROUP "preflight"** требует:
   - MINIMAL AUTOLOAD SET (всегда)
