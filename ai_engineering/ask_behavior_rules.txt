# Default ASK Behavior Rules

При обработке любого запроса:

1. Формируй короткий, безопасный, атомарный план (Plan) без выполнения.
2. Не выполняй шаги самостоятельно — только планируй.
3. Всегда запрашивай подтверждение пользователя перед передачей плана в Agent.
4. Не сканируй проект, не читай директории, не расширяй контекст.
5. Работай только с файлами, которые указаны пользователем.
6. Соблюдай принципы ARCHITECTURE_GUIDE.md:
   - идемпотентность
   - атомарность
   - KISS
   - documented decisions
   - clean root
   - modular isolation
7. Уважай CONTEXT_RULES.md (Controlled Context Mode).
8. Соблюдай стиль AI_WORKSTYLE_GUIDE.md (интеллектуальный партнёр + shift expectations).
9. Перед планированием всегда учитывать ai_engineering/AI_ISSUES_LOG.md.
10. **ОБЯЗАТЕЛЬНАЯ ПРОВЕРКА MECHANISM_HIERARCHY.md:**
    - Перед предложением ЛЮБОГО нового механизма (validation, safety, guard, protocol, enforcement и т.д.) ОБЯЗАТЕЛЬНО проверь `ai_engineering/MECHANISM_HIERARCHY.md`.
    - Если PRIMARY механизм для области логики уже существует — создание нового ЗАПРЕЩЕНО.
    - Нарушение этого правила считается архитектурной ошибкой.
    - Если PRIMARY не существует — создай новый и НЕМЕДЛЕННО обнови MECHANISM_HIERARCHY.md.
11. **КОНТЕКСТ-РОУТИНГ (ОБЯЗАТЕЛЬНО):**
    - Используй `ai_engineering/AI_CONTEXT_ROUTER.md` для выбора правил по задаче.
    - Загружай дополнительные правила ТОЛЬКО через роутер по категории задачи (planning, execution, preflight, debug, architecture и т.д.).
    - ЗАПРЕЩЕНО загружать все правила подряд или читать директории `ai_engineering/` без явного указания файла.
    - Навигация по правилам осуществляется через `ai_engineering/AI_RULES_INDEX.md`.
    - Это дефолтный режим "правила работы" — применяется автоматически без напоминаний пользователя.
