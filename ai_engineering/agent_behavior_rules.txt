# Agent Behavior Rules (Safety Wrapper)

Эти правила обязательны для выполнения любого плана в режиме AGENT.

1. Выполняй только те изменения, которые указаны в плане, сформированном ASK.
2. Не изменяй файлы, не перечисленные в плане.
3. Не сканируй проект, не читай директории, не загружай новые файлы.
4. Не выполняй "догадочные" исправления, только явные шаги.
5. Все изменения должны быть:
   - атомарными,
   - идемпотентными,
   - минимальными,
   - обратимыми,
   - строго соответствующими плану.
6. Если план кажется потенциально опасным — запроси подтверждение пользователя.
7. Если шаг невозможно выполнить — сообщи об ошибке, но не выдумывай альтернативу.
8. Уважай архитектурные ограничения (ARCHITECTURE_GUIDE.md).
9. Соблюдай Controlled Context Mode (CONTEXT_RULES.md).
10. Соблюдай стиль работы AI_WORKSTYLE_GUIDE.md — быть инженером-партнёром.
11. Логику не улучшай самостоятельно — только выполняй шаги.
12. Никогда не выполняй массовые правки, если этого нет в плане.
13. Любое изменение должно быть прозрачно отражено в diff/patch.
14. НИКОГДА не создавай и не редактируй файлы через терминал (echo/printf/heredoc). Всегда используй `write_file` или `apply_patch` для файловых операций; терминал предназначен только для запуска команд.
15. СТРОГО ЗАПРЕЩЕНО вызывать блокирующие процессы через run_terminal_cmd:
    - ssh.exe / scp.exe — используй `infrastructure/scripts/Invoke-Safe*.ps1`
    - xray.exe / wireguard.exe — используй `infrastructure/safe-run/Start-*.ps1`
    - любые туннели/демоны — только через Start-Process с `-WindowStyle Hidden`
    REASON: Прямые вызовы блокируют tool protocol и вызывают deadlock агента.
16. STRICTLY FORBIDDEN:
    - ssh-keygen (любые варианты запуска)
    REASON:
      Команда запрашивает интерактивный ввод и не может быть выполнена безопасно через run_terminal_cmd.
      Всегда запускается вручную пользователем или через безопасный внешний скрипт.
17. Для файлов >200 строк используй точечное чтение:
    - `grep` для поиска по шаблону;
    - `read_file` с `offset/limit` для чтения диапазона.
    Полное чтение больших файлов засоряет контекст и замедляет работу.

18. **Definition of Done — обязательные артефакты после выполнения:**
    После выполнения ЛЮБОГО изменения Agent ОБЯЗАН предоставить:
    
    a) **Краткое описание сделанного:**
       - Что было изменено (краткое резюме)
       - Какие файлы затронуты
    
    b) **Список изменённых файлов:**
       - Полные пути к изменённым файлам
       - Количество изменённых строк (добавлено/удалено, если доступно)
    
    c) **Diff/Patch summary:**
       - Краткое описание изменений в каждом файле
       - Прозрачное отражение всех изменений
    
    d) **Результат проверки / smoke-test (если применимо):**
       - Если изменён код — результат запуска теста или проверки работоспособности
       - Если изменён скрипт — результат пробного запуска (если безопасно)
       - Если изменена конфигурация — результат валидации (например, через Lint-ConfigFiles.ps1 для JSON/CONF)
       - Статус проверки (OK/FAIL) с кратким выводом
    
    e) **Итоговый статус:**
       - Success (все изменения применены, проверки пройдены)
       - Partial (часть изменений применена, требуется доработка)
       - Failed (изменения не применены или проверки не пройдены)
       - Следующие шаги (если требуются)
    
    **ЗАПРЕТ:** Завершение выполнения БЕЗ предоставления артефактов (a-e) считается нарушением Definition of Done.
    
    **ИСКЛЮЧЕНИЯ:**
    - Если выполнение было заблокировано Agent Safety Executor — достаточно сообщения о блокировке
    - Если выполнение было прервано Error Protocol — достаточно сообщения об ошибке и ссылки на AI_ISSUES_LOG.md

== END RULES ==
