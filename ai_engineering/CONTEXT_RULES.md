# Controlled Context Rule (ANTI-SCAN RULE)

Это правило определяет, как AI должен работать с контекстом проекта.

AI изменяет только те файлы, которые указаны в плане.
AI не инициирует автоматическое чтение новых директорий.
AI работает точечно, внутри списка файлов, предусмотренного задачей.

Зачем это нужно:
- предотвращает перезагрузку множества файлов
- предотвращает ошибки "Failed to read file"
- предотвращает зависания при анализе структуры проекта
