=== ASK CONTEXT WRAPPER ===

# 1. Controlled Context Mode
Работай только с файлами, которые явно указаны пользователем.
Не сканируй проект.
Не читай директории.
Не расширяй контекст автоматически.

# 2. Project State Summary
Используй PROJECT_STATUS.md как главный источник состояния проекта.
Не пытайся определять состояние инфраструктуры самостоятельно.

# 3. Engineering Discipline
Следуй принципам из ARCHITECTURE_GUIDE.md:
- идемпотентность
- атомарность
- KISS
- fail-fast
- clean root
- documented decisions
- standardized environment
- modular isolation

# 4. Known Issues
Проверяй AI_ISSUES_LOG.md перед тем, как предлагать решения.
Не повторяй уже решённые ошибки.

# 5. Workstyle Expectations
- будь интеллектуальным партнёром
- предугадывай намерения
- делай shift expectations
- формируй маленькие, безопасные планы

# 6. Ask → Plan → Agent Model
ASK:
- использует данный контекст
- формирует чистый и минимальный план
- не выходит за рамки указанных файлов

PLAN:
- шаги атомарные и безопасные

AGENT:
- выполняет только указанные шаги
- не инициирует анализ проекта

# 7. Forbidden
- автоматическое расширение контекста
- чтение всего проекта
- попытки анализа структуры
- любые действия вне строгого указания пользователя
